## 🚀 The Problem
Health Technology Assessment (HTA) bodies often rely on meta-analyses that are:
1.  **Biased:** Skewed by missing small studies (Publication Bias).
2.  **Premature:** Statistically significant but underpowered (Information Size Crisis).
3.  **Fragile:** Conclusions that flip with the removal of a single study.

## 🛠️ The Solution
This toolkit provides a systematic pipeline to **filter, correct, and predict** the integrity of Cochrane evidence.

### 1. The Core Pipeline
Run these scripts in order to generate your HTA quality report:

1.  `analysis/bias_correction_pipeline.R`
    *   **What it does:** Systematically runs Egger's Test and Trim-and-Fill on all datasets.
    *   **Output:** `bias_corrected_results.csv` (Original vs. Corrected estimates).

2.  `analysis/unified_hta_validation.R`
    *   **What it does:** Calculates Information Fraction (IF) and combines it with bias checks.
    *   **Output:** `unified_hta_validation.csv` (Classifies evidence as "Gold Standard", "Premature", etc.).

3.  `analysis/predict_evidence_decay.R`
    *   **What it does:** Uses Machine Learning to predict which "Premature" findings are 100% likely to fail in future updates.
    *   **Output:** `evidence_decay_predictions.csv`.

### 2. The Decision Dashboard
Open **`HTA_Dashboard.html`** in your browser.
*   **Visualizes:** The "Evidence Quality Landscape".
*   **Watchlist:** Identifies the specific Cochrane reviews that are "High Risk" for evidence decay.
